# 🔄 Протокол: Self-Annealing

> **Мета:** Коли скрипт або MCP tool повертає помилку — виправити код, а не "перепромпчувати".
> **Чому:** "Спробувати ще раз" без змін = та сама помилка. Виправлення має бути в коді.

---

## Тригер

Цей протокол активується коли:
- PowerShell скрипт з `execution/` повернув `exit code 1`
- MCP tool повернув помилку
- Команда `npx firebase deploy` зафейлилась
- Будь-який детермінований інструмент не спрацював

## Алгоритм

```
┌──────────────────────────────────────────┐
│ 1. DIAGNOSE: Прочитай повний error output│
│    - Яка КОНКРЕТНА помилка?              │
│    - В якому ФАЙЛІ?                      │
│    - На якому РЯДКУ?                     │
├──────────────────────────────────────────┤
│ 2. CLASSIFY: Визнач тип помилки          │
│    ├── A) Баг в скрипті execution/       │
│    ├── B) Баг в rules/коді               │
│    ├── C) Зовнішня проблема (мережа/API) │
│    └── D) Невірна директива              │
├──────────────────────────────────────────┤
│ 3. FIX: Виправ КОНКРЕТНИЙ файл           │
│    A) → Змінити скрипт в execution/      │
│    B) → Змінити rules/код проекту        │
│    C) → Додати retry логіку в скрипт     │
│    D) → Оновити директиву в directives/  │
├──────────────────────────────────────────┤
│ 4. UPDATE: Оновити ДОКУМЕНТАЦІЮ          │
│    - Директива: додати "Відомі помилки"  │
│    - Скрипт: додати коментар з fix       │
├──────────────────────────────────────────┤
│ 5. RE-RUN: Виконати виправлений скрипт   │
│    - Якщо знову fail → повернутись до 1  │
│    - Максимум 3 ітерації                 │
├──────────────────────────────────────────┤
│ 6. ESCALATE: Якщо 3 ітерації не допомогли│
│    → Повідомити користувача              │
│    → Показати всі 3 спроби та помилки    │
│    → Запропонувати альтернативний підхід  │
└──────────────────────────────────────────┘
```

## Заборони

| ❌ ЗАБОРОНЕНО | ✅ ПРАВИЛЬНО |
|--------------|-------------|
| Просто запустити скрипт ще раз | Прочитати помилку, виправити причину |
| Ігнорувати помилку | Класифікувати та виправити |
| "Давайте спробуємо інший підхід" без аналізу | Зрозуміти ЧОМУ зафейлилось |
| Видалити скрипт і написати новий | Виправити існуючий скрипт |
| Перепромпчувати те саме | Змінити код або директиву |

## Приклад

### Помилка
```
❌ validate-rules.ps1 exit code 1
   "firebase\firestore.rules — НЕ ЗНАЙДЕНО"
```

### Self-Annealing
```
1. DIAGNOSE: Файл firestore.rules не знайдено. Шлях невірний.
2. CLASSIFY: A) Баг в скрипті — невірний шлях
3. FIX: Змінити шлях з "firebase\" на "firebase/" або перевірити $ProjectRoot
4. UPDATE: Додати коментар у validate-rules.ps1
5. RE-RUN: Запустити знову → PASS
```

## Метрика

Відстежувати в коментарях скрипта:
```powershell
# SELF-ANNEALING LOG:
# [2026-04-19] Fix: шлях до rules файлу (backslash → правильний Join-Path)
# [2026-04-20] Fix: додано fallback для відсутнього firebase.json
```
