# 🔧 Директива: Додати новий скрипт виконання

> **Мета:** Створити новий інструмент (скрипт) та його документацію.
> **Чому:** Кожен скрипт ПОВИНЕН мати директиву. Інакше наступний агент не знатиме коли та як його використовувати.

---

## Коли застосовувати

- Агенту потрібен інструмент якого немає в `execution/`
- Користувач просить автоматизувати повторювану задачу
- Існуючий скрипт не покриває новий use case

## Процедура

### Крок 1: Визначити потребу

Що саме потрібно автоматизувати?
- API виклик → PowerShell скрипт
- Firebase операція → MCP tool (не потребує скрипту)
- Обробка даних → PowerShell скрипт
- Валідація → PowerShell скрипт

### Крок 2: Створити скрипт в `execution/`

**Файл:** `.agents/execution/<назва-кебаб>.ps1`

**Вимоги до скрипту:**
```powershell
<#
.SYNOPSIS
    Короткий опис що робить скрипт

.DESCRIPTION
    Детальний опис. Які дані читає, які перевірки робить.

.PARAMETER WhatIf
    Dry-run режим — показує що БУДЕ зроблено, але не змінює нічого.

.EXAMPLE
    .\execution\my-script.ps1
    .\execution\my-script.ps1 -WhatIf
#>

param(
    [switch]$WhatIf
)

# --- Конфігурація ---
$ErrorActionPreference = "Stop"
$ProjectRoot = Split-Path -Parent (Split-Path -Parent $PSScriptRoot)

# --- Чутливі дані --- 
# Завантажити з .env якщо потрібно:
# $envFile = Join-Path $PSScriptRoot "..\env\.env"
# if (Test-Path $envFile) { Get-Content $envFile | ForEach-Object { ... } }

# --- Логіка ---
# ...

# --- Результат ---
# Exit 0 = success, Exit 1 = failure
```

### Крок 3: Створити директиву в `directives/`

**Файл:** `.agents/directives/<назва-кебаб>.md`

**Шаблон:**
```markdown
# [Emoji] Директива: [Назва]

> **Мета:** ...
> **Чому:** ...

## Передумови
- Тригер: ...

## Процедура
### Крок 1: ...
**Виконати:** `execution/<назва>.ps1`
### Крок 2: ...

## Пов'язані ресурси
| Тип | Шлях |
|-----|------|
| Скрипт | `.agents/execution/<назва>.ps1` |

## При помилці
→ Застосувати протокол `protocols/self-annealing.md`
```

### Крок 4: Оновити реєстр

Додати новий рядок в `directives/_index.md`:
```markdown
| "<тригер>" | [<назва>.md](<назва>.md) | — |
```

### Крок 5: Протестувати

```powershell
# Dry-run
.\execution\<назва>.ps1 -WhatIf
```

---

## Правила

1. **ЗАВЖДИ парно:** 1 скрипт = 1 директива
2. **Скрипт не хардкодить** API ключі → читає з `env/.env`
3. **Скрипт підтримує** `-WhatIf` для dry-run
4. **Директива пояснює** "що" і "чому", скрипт робить "як"
5. **Реєстр оновлений** — `_index.md` містить новий запис
